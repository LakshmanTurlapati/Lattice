    Implement the public runtime facade.

    In `runtime/create-ai.ts`:
    - Export `RunIntent<TOutputs extends OutputContractMap>` with readonly `task: string`, optional readonly `artifacts?: readonly ArtifactInput[]`, readonly `outputs: TOutputs`, optional readonly `policy?: PolicySpec`, optional readonly `session?: SessionRef`, and optional readonly `signal?: AbortSignal`.
    - Export `AI` with `run<const TOutputs extends OutputContractMap>(intent: RunIntent<TOutputs>): Promise<RunResult<TOutputs>>`.
    - Export `createAI(config: LatticeConfig = {}): AI`.
    - Use `normalizeConfig(config)` once inside `createAI`.
    - In `run`, if `intent.signal?.aborted` is true, throw `new DOMException("Run aborted before execution.", "AbortError")`.
    - Create a plan stub for every run using `createExecutionPlanStub()`.
    - Merge policy using `mergePolicy(normalized.defaults.policy, intent.policy)`.
    - Select the first configured provider entry where `kind === "provider-adapter"` and `typeof execute === "function"`.
    - If no executable adapter exists, return `{ ok: false, error: { kind: "execution_unavailable", message: "No Phase 1 provider adapter with execute() is configured." }, plan }`.
    - Call the adapter with a Lattice-owned request containing `task`, `artifacts: intent.artifacts ?? []`, `outputs: Object.keys(intent.outputs)`, `policy: mergedPolicy`, and `signal: intent.signal`.
    - Pass `response.rawOutputs` to `validateOutputMap(intent.outputs, response.rawOutputs, plan)`.
    - If validation succeeds, return the validation success with `artifacts: response.artifactRefs` narrowed to `readonly ArtifactRef[]` when present, otherwise `[]`.
    - If validation fails, return the validation failure unchanged except ensure it carries the same plan.
    - If adapter execution throws, return `{ ok: false, error: { kind: "execution_unavailable", message: error instanceof Error ? error.message : "Provider adapter execution failed." }, plan }`.
    - Do not import provider SDKs or implement routing/fallback/provider packaging.

    In `runtime/public-types.ts`:
    - Re-export public types only: `AI`, `RunIntent`, `LatticeConfig`, `NormalizedLatticeConfig`, `ProviderAdapter`, `ProviderRef`, `ProviderRunRequest`, `ProviderRunResponse`, `PolicySpec`, `StorageLike`, `TracerLike`, `SessionRef`, `ExecutionPlanStub`, `ArtifactInput`, `ArtifactRef`, `ArtifactKind`, `OutputContract`, `OutputContractMap`, `InferOutput`, `InferOutputMap`, `RunResult`, `RunSuccess`, `RunFailure`, `LatticeRunError`, `ValidationIssue`.

    In `test/runtime.test.ts`, add Vitest runtime tests for fixture adapter success, validation failure, execution unavailable, and default policy override. The fixture adapter must be a Lattice `ProviderAdapter`, not an OpenAI/AI SDK object.
